## Trust Boundaries

|----------|-------------|
| cliente HTTP → ProdutoController | Query params (descricao, codigobarra, iddepartamento, idgrupo, idmarca, page, take) e path param idproduto cruzam não confiáveis |
| AthosService → banco Athos PostgreSQL | Queries SQL contra o banco externo read-only (192.168.3.198) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Tampering (SQL Injection) | buscarProdutos / buscarProdutoPorId / lookups em AthosService | mitigate | Todos os valores de filtro passam por placeholders parametrizados ($1, $2, ...) via `client.query(sql, params)`. Nenhuma interpolação de string de valores na SQL. idproduto validado por ParseIntPipe (só aceita inteiro) antes de chegar ao service. |
| T-32-02 | Information Disclosure | coluna imagemproduto (bytea) na resposta de busca/id | mitigate | Queries usam `SELECT p.*, NULL::bytea AS imagemproduto` — a coluna binária nunca é serializada para o JSON de resposta. Interface `Produto` tipa `imagemproduto: null`. |
| T-32-03 | Denial of Service | listagem sem filtro sobre 28.836 produtos | mitigate | `take` capado em `Math.min(..., 50)` e mínimo `Math.max(1, ...)`; paginação obrigatória com LIMIT/OFFSET. ThrottlerGuard global já aplicado ao app. |
| T-32-04 | Spoofing / Elevation (auth bypass) | endpoints /athos/produtos sem autenticação | mitigate | ProdutoController NÃO reimplementa auth; herda o APP_GUARD global (InternalAuthGuard via x-internal-api-key, timingSafeEqual). Requisição sem chave válida → 401. Garantido por não injetar nenhuma rota fora do pipeline do guard. |
| T-32-SC | Tampering | npm/pip/cargo installs | accept | Nenhum pacote novo é instalado nesta fase (Package Legitimacy Audit do RESEARCH.md: "Não aplicável"). Todos os recursos (@nestjs/common, @nestjs/swagger, pg) já presentes no projeto. |
