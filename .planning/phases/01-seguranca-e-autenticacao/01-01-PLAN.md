<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/proxy -> backend api | entrada nao confiavel cruza para regras de negocio |
| efi webhook -> backend | payload financeiro externo pode ser forjado |
| env/processo -> bootstrap | configuracao invalida pode deixar sistema inseguro |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S | API interna sem auth | mitigate | aplicar `APP_GUARD` com `x-internal-api-key` e `@Public` apenas em excecoes |
| T-01-02 | I | CORS aberto | mitigate | configurar allowlist de origem em `main.ts` via env |
| T-01-03 | E | bypass involuntario de auth | mitigate | metadata explicita `isPublic` e guard deny-by-default |
| T-01-04 | D | bootstrap com env faltando | mitigate | validacao de env no startup com falha imediata |
</threat_model>
