## Riscos aceitos (ACCEPTED — 1)

### T-999.1-SANDBOX — Chromium `--no-sandbox` como root renderizando HTML não confiável
- **Categoria:** Elevation of Privilege
- **Decisão:** **Aceito** para o cenário de deploy interno.
- **Justificativa / controles compensatórios:**
  - Rede 100% bloqueada no Puppeteer (anti-SSRF) — `shouldAllowRequest` + `setRequestInterception`.
  - Upload sanitizado: `validateUpload` rejeita `<script>`, `on*=`, `javascript:`, `iframe`, etc.
  - Compilação Handlebars restrita (`knownHelpersOnly`, instância isolada).
  - Deploy em container isolado, sem exposição pública.
- **Dívida para endurecer (backlog):** usuário não-root no Dockerfile + perfil seccomp do Chromium.

## Ação obrigatória pré-deploy (não bloqueia o fechamento da fase)

> ⚠️ **CR-01:** o painel admin é *fail-open*. Em produção, **defina** `ADMIN_API_KEY`,
> `CONFIG_PANEL_PASSWORD` e `CONFIG_PANEL_SESSION_SECRET` (valores fortes, distintos) — e
> adicione-os ao `deploy/stack.env.example`. Sem isso, o painel sobe sem proteção.
