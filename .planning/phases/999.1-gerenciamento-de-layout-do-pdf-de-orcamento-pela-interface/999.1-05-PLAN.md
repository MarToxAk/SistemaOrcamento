<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cliente -> POST upload | source de template e input nao confiavel |
| admin handler -> Prisma | mutacoes so apos AdminAuthGuard |
| preview -> render hardened | preview executa HTML nao confiavel (mesmo risco do render normal) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-999.1-14 | Tampering | XSS armazenado no template (<script>, on*=) | mitigate | validateUpload REJEITA (nao limpa) HTML perigoso server-side (V5; Open Question 3) |
| T-999.1-15 | Denial of Service | upload gigante | mitigate | limite de 512KB no validateUpload |
| T-999.1-16 | Spoofing | acesso aos endpoints de escrita/preview | mitigate | @AdminOnly() (AdminAuthGuard do Plano 02) em todos os handlers |
| T-999.1-17 | Elevation of Privilege | preview renderizando HTML nao confiavel sem hardening | mitigate | preview usa o render hardened do Plano 03 (compile restrito + rede bloqueada) |
| T-999.1-SC | Tampering | npm install sanitize-html | mitigate | checkpoint:human-verify bloqueante antes do install (flag SUS de recencia) |
</threat_model>
