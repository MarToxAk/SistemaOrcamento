<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser admin -> Next route handler | requisicao do admin (sem segredo no cliente) |
| Next route handler -> backend | proxy injeta x-admin-api-key server-side (Plano 06) |
| backend handler de escrita | AdminAuthGuard valida a chave antes de qualquer mutacao |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-999.1-03 | Spoofing | endpoints de escrita de template | mitigate | AdminAuthGuard com x-admin-api-key dedicado, timingSafeEqual; 401 quando chave ausente |
| T-999.1-04 | Information Disclosure | timing attack na comparacao da chave | mitigate | crypto.timingSafeEqual com checagem previa de tamanho (Don't Hand-Roll do RESEARCH) |
| T-999.1-05 | Information Disclosure | segredo de admin vazando ao browser | mitigate | proibir NEXT_PUBLIC_; chave so server-side, injetada por proxy (Pitfall 5; Plano 06) |
| T-999.1-06 | Elevation of Privilege | reuso de INTERNAL_API_KEY como admin | mitigate | credencial dedicada ADMIN_API_KEY (D-03), guard separado, NAO global |
</threat_model>
