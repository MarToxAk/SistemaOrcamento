## Schema Additions

```
enum PdfTemplateKind { PRESET CUSTOM }

model PdfTemplate {
  id        String          @id @default(cuid())
  name      String
  slug      String          @unique
  source    String          @db.Text
  kind      PdfTemplateKind
  isActive  Boolean         @default(false)
  createdAt DateTime        @default(now())
  updatedAt DateTime        @updatedAt
  @@index([isActive])
}
```

- **Fix:** Verificado que os tipos TypeScript (`.d.ts`) foram atualizados com sucesso; `grep PdfTemplate node_modules/.prisma/client/index.d.ts` confirmou `PdfTemplate` e `PdfTemplateKind` presentes. A DLL binária já estava na versão correta — erro foi apenas no rename temporário
- **Commit:** 2942b40

## Operator Instructions

O operador deve aplicar a migration manualmente após o próximo deploy:

```bash
DATABASE_URL=<connection-string> npx prisma migrate deploy
```

O seed roda automaticamente via `bootstrap-runtime.sh` após o migrate deploy. Se `ts-node` não estiver disponível no container, rodar manualmente:

```bash
npx ts-node --project apps/backend/tsconfig.json apps/backend/scripts/seed-pdf-templates.ts
```
