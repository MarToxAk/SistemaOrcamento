## Goal

Completar o `AthosListenerService` com reconexão automática (CAIXA-01), notificação Chatwoot ao detectar pagamento no caixa (CAIXA-02), proteção do endpoint SSE (CAIXA-03) e cobertura de testes unitários (CAIXA-04).

## Context Summary

O serviço `athos-listener.service.ts` já substitui o `bridge.js`/n8n e está rodando em produção. O core está funcional (LISTEN, query, SSE, paymentNote). O que falta é o hardening: reconexão, Chatwoot e testes.

Ver análise completa: `23-CONTEXT.md`.

## Threat Model

| Risco | Mitigação |
|-------|-----------|
| PG restart silencia o listener indefinidamente | Reconnect com backoff exponencial (CAIXA-01) |
| SSE expõe dados de pagamento sem auth | Guard por `x-internal-api-key` (CAIXA-03) |

## Waves

### Wave 1 — Reconexão automática (CAIXA-01)

**Objetivo:** Se o `pg.Client` cair, reconectar com backoff exponencial (1s → 2s → 4s → 8s → máx 30s).

**Arquivo:** `apps/backend/src/modules/integrations/athos/athos-listener.service.ts`

**Mudanças:**

1. Extrair a lógica de conexão para um método privado `connect()`.
2. No handler `client.on("error")`, marcar `this.client = null` e chamar `scheduleReconnect()`.
3. Implementar `scheduleReconnect(attempt: number)` com `setTimeout` e delay `Math.min(2^attempt * 1000, 30_000)`.
4. Limpar o timer no `onApplicationShutdown` para evitar reconnect após shutdown.

**Pseudocódigo:**

```typescript
private reconnectAttempt = 0;
private reconnectTimer: ReturnType<typeof setTimeout> | null = null;

private async connect(): Promise<void> {
  const cfg = buildConfig();
  this.client = new Client({ ...cfg, connectionTimeoutMillis: 10_000 });
  await this.client.connect();
  await this.client.query(`LISTEN ${LISTEN_CHANNEL}`);
  this.reconnectAttempt = 0; // reset on success

  this.client.on("notification", (msg) => { void this.handleNotification(); });
  this.client.on("error", (err) => {
    this.logger.error(`Listener error: ${err.message}`);
    this.client = null;
    this.scheduleReconnect();
  });
}

private scheduleReconnect(): void {
  const delay = Math.min(Math.pow(2, this.reconnectAttempt) * 1000, 30_000);
  this.reconnectAttempt++;
  this.logger.warn(`Reconectando em ${delay}ms (tentativa ${this.reconnectAttempt})`);
  this.reconnectTimer = setTimeout(() => void this.connect(), delay);
}
```

**Commit:** `fix(athos-listener): reconexão automática com backoff exponencial`

### Wave 2 — Notificação Chatwoot ao pagar no caixa (CAIXA-02)

**Objetivo:** Quando `handleNotification()` detectar pagamento no caixa, buscar `conversationId` do orçamento no Prisma e enviar mensagem Chatwoot (fire-and-forget).

**Arquivos:**
- `apps/backend/src/modules/integrations/athos/athos-listener.service.ts`
- `apps/backend/src/modules/integrations/athos/athos.module.ts`

**Mudanças:**

1. Em `athos.module.ts`: adicionar `ChatwootModule` ao array `imports` (já exporta `ChatwootService`).
2. Em `AthosListenerService`: injetar `ChatwootService` no construtor.
3. Após `persistPaymentNote`, chamar `notifyChatwoot(idorcamento)` (fire-and-forget, sem await no fluxo principal).

**Implementação `notifyChatwoot`:**

```typescript
private notifyChatwoot(idorcamento: number, numeroordem: string): void {
  void (async () => {
    try {
      const quote = await this.prisma.quote.findFirst({
        where: { externalQuoteId: BigInt(idorcamento) },
        select: { conversationId: true },
      });
      if (!quote?.conversationId) return;

      await this.chatwootService.sendOutgoingMessage(
        String(quote.conversationId),
        `Pagamento confirmado no caixa — Pedido #${numeroordem}. Obrigado!`,
      );
      this.logger.log(`Chatwoot notificado: conversationId=${quote.conversationId}`);
    } catch (err) {
      const msg = err instanceof Error ? err.message : String(err);
      this.logger.warn(`Falha ao notificar Chatwoot (caixa): ${msg}`);
    }
  })();
}
```

**Ponto de chamada** (em `handleNotification`, após `persistPaymentNote`):
```typescript
this.notifyChatwoot(idorcamento, numeroordem);
// nota: sem await — não bloqueia listener
```

**Commit:** `feat(athos-listener): notificação Chatwoot ao detectar pagamento no caixa`

### Wave 3 — Proteger SSE endpoint (CAIXA-03)

**Objetivo:** Remover `@Public()` do `GET /api/events/pagamentos` e proteger com o guard de API key existente.

**Arquivo:** `apps/backend/src/modules/events/events.controller.ts`

**Mudança:**

```typescript
// Antes:
@Public()
@Sse("pagamentos")
streamPagamentos()

// Depois:
@Sse("pagamentos")
streamPagamentos()
```

Remover `@Public()` — o guard global `x-internal-api-key` passará a valer.

**Impacto no frontend:** A rota proxy Next.js que consome o SSE precisa passar o header. Verificar `apps/frontend/src/app/api/events/pagamentos/route.ts` (ou similar). Se não existir, criar o proxy.

**Verificação de proxy SSE no frontend:**
```bash
# Verificar se existe rota proxy para o SSE
ls apps/frontend/src/app/api/events/
```

Se não existir, criar `apps/frontend/src/app/api/events/pagamentos/route.ts`:
```typescript
export async function GET() {
  const backendUrl = process.env.BACKEND_URL ?? "http://localhost:4000";
  const apiKey = process.env.INTERNAL_API_KEY ?? "";
  const upstream = await fetch(`${backendUrl}/api/events/pagamentos`, {
    headers: { "x-internal-api-key": apiKey },
  });
  return new Response(upstream.body, {
    headers: {
      "Content-Type": "text/event-stream",
      "Cache-Control": "no-cache",
      "Connection": "keep-alive",
    },
  });
}
```

**Commit:** `fix(events): proteger SSE /pagamentos com x-internal-api-key`

### Wave 4 — Testes unitários (CAIXA-04)

**Objetivo:** Cobrir `AthosListenerService` com testes Jest.

**Arquivo:** `apps/backend/src/modules/integrations/athos/athos-listener.service.test.ts`

**Cenários a cobrir:**

| Cenário | Assertion |
|---------|-----------|
| Caixa detectado | `persistPaymentNote` chamado, SSE emitido, Chatwoot notificado |
| Venda sem `idcaixamovimento` | Retorna sem chamar `persistPaymentNote` |
| `relacao_orcamento_venda` vazia | Retorna sem erro |
| `persistPaymentNote` lança exceção | Erro logado, SSE ainda emitido |
| Reconexão após erro | `connect()` chamado novamente após delay |
| Chatwoot sem `conversationId` | Não chama `sendOutgoingMessage` |

**Mock strategy:**
- Mock `pg.Client` com `jest.fn()` para `connect`, `query`, `on`
- Mock `PrismaService.quote.updateMany` e `findFirst`
- Mock `EventsService.emitCaixaPayment`
- Mock `ChatwootService.sendOutgoingMessage`

**Commit:** `test(athos-listener): cobertura de reconexão, caixa e Chatwoot`
