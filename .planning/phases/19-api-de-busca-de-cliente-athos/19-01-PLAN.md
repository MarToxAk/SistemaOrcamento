  <action>
Adicionar metodo `buscarClientes(params)` no AthosService, sem remover os metodos atuais.

Regras minimas do metodo:
1. Aceitar filtros: `nome?`, `documento?`, `idcliente?`, `page?`, `take?`
2. Validar `take` entre 1 e 50 (default 20) e `page >= 1`
3. Exigir ao menos um filtro significativo (`nome` com >= 3 chars, ou `documento`, ou `idcliente`)
4. Consultar base Athos em modo read-only usando join entre:
   - cliente c
   - cliente_fisico cf (left join)
   - cliente_juridico cj (left join)
   - cliente_endereco ce (left join limitado ao primeiro endereco por cliente)
5. Normalizar cada item para:
   - idcliente
   - tipoPessoa: "fisico" | "juridico"
   - nome (cf.nome ou cj.nomefantasia/razaosocial)
   - documento (cpf/cnpj sem mascara)
   - endereco resumido (quando existir)
6. Retornar objeto paginado: `{ total, page, take, items }`
7. Em caso de erro de query, logar com `this.logger.warn` e relancar `InternalServerErrorException` com mensagem clara

Observacoes:
- Preferir SQL parametrizado (sem concatenar input bruto)
- Nao alterar comportamento de `buscarClientePorId`
  </action>
