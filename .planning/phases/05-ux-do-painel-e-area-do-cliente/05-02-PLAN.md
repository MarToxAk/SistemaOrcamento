  <action>
Editar `apps/frontend/src/app/orcamento/novo/page.tsx`:

**1. Adicionar estado `submitted`** (junto aos outros useState no topo do componente):
```tsx
const [submitted, setSubmitted] = useState(false);
```

**2. Atualizar a classe do `<form>`** para incluir `was-validated` condicionalmente:
```tsx
<form
  ref={formRef}
  className={`needs-validation${submitted ? " was-validated" : ""}`}
  noValidate
  onSubmit={handleSubmit}
>
```
Se o form usa `onSubmit` em outro lugar, ajustar onde o handler é referenciado.

**3. Atualizar o submit handler** para chamar `setSubmitted(true)` ANTES de `formRef.current?.checkValidity()`:
```tsx
setSubmitted(true);
if (formRef.current && !formRef.current.checkValidity()) {
  return; // Bootstrap CSS mostra erros automaticamente via CSS :invalid
}
// continuar com o submit normal
```
Remover qualquer lógica de validação manual que duplique este comportamento (mas manter `itensInvalidos` check se existir, pois é validação de negócio não cobertura pelo HTML5).

**4. Adicionar `<div className="invalid-feedback">` após cada `<input required>`**:
Localizar cada campo obrigatório pelo `id` e adicionar imediatamente após o `<input>` (não após o label):

```tsx
// Após <input id="clienteNome" required ... >
<div className="invalid-feedback">Nome do cliente é obrigatório.</div>

// Após <input id="telefone" required ... >
<div className="invalid-feedback">Telefone é obrigatório.</div>

// Após <input id="vendedor" required ... >
<div className="invalid-feedback">Informe o nome do vendedor.</div>

// Após <input id="validade" required ... >
<div className="invalid-feedback">Informe a validade da proposta.</div>

// Após <input id="prazoEntrega" required ... >
<div className="invalid-feedback">Informe o prazo de entrega.</div>
```

**5. Adicionar asterisco vermelho nos labels dos campos obrigatórios**:
```tsx
// Exemplo:
<label htmlFor="clienteNome">Nome do Cliente <span className="text-danger" aria-hidden="true">*</span></label>
```
Aplicar o mesmo padrão para `telefone`, `vendedor`, `validade` e `prazoEntrega`.

**6. Suprimir bordas verdes em campos válidos** — adicionar CSS inline (via `<style>` tag ou via `globals.css` se disponível).
A forma mais simples é inserir um `<style>` no JSX do componente (aceitável pois é um formulário isolado):
```tsx
<style>{`
  .needs-validation.was-validated .form-control:valid,
  .needs-validation.was-validated .form-select:valid {
    border-color: #ced4da;
    background-image: none;
  }
`}</style>
```
Inserir este `<style>` antes do `<form>` ou dentro do fragment.

**IMPORTANTE**: Não alterar a lógica de envio para o servidor, não alterar o `DADOS_EXEMPLO`, não alterar o cálculo de carimbos ou itens. Modificar apenas a lógica de validação visual.
