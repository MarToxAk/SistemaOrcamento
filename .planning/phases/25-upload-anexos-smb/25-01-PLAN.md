## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API multipart | nome do arquivo, tamanho, extensao e campos auxiliares entram como input nao confiavel |
| API -> SMB share | gravacao fisica em UNC path pode sofrer traversal, nome invalido ou indisponibilidade de rede |
| API -> Athos PostgreSQL | persistencia do metadado do anexo deve permanecer parametrizada e consistente com o arquivo salvo |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | T | endpoint upload / nome do arquivo | mitigate | Sanitizar com `path.win32.basename`, whitelist de extensoes e rejeicao de separadores, `..` e caracteres reservados |
| T-25-02 | D | multipart parsing | mitigate | Limitar a 10MB com validator no `@UploadedFile()` e falhar antes de qualquer escrita em disco/share |
| T-25-03 | T | service gravacao + insert | mitigate | Ordem deterministica de operacoes e cleanup compensatorio (`unlink` best-effort) se o INSERT falhar apos gravacao fisica |
| T-25-04 | S | auth do endpoint novo | mitigate | Reusar `validateAthosToken()` fail-closed da fase 24 em vez de nova logica paralela |
| T-25-05 | I | mensagens/logs de infraestrutura SMB | accept | Log detalhado apenas no backend; resposta HTTP ao cliente deve ser clara sem vazar credenciais ou internals do host |
