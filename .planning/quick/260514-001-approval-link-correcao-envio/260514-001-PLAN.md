# Quick Task 260514-001: Correção Link de Aprovação no Envio de Orçamentos

## Goal

Corrigir o bug onde o link de aprovação nunca é incluído na mensagem enviada ao cliente, mesmo quando o `idcliente` Athos é encontrado.

## Root Cause

Em `enviarParaCliente` (quotes.service.ts), o approval link só é gerado quando:
```
if (clienteId && isAssociatedCustomer)
```

`isAssociatedCustomer` vem de `quote.customer.isAssociated`, que tem valor padrão `false` e nunca é alterado para `true` automaticamente ao importar orçamentos do Athos. Mesmo que `clienteId` seja resolvido com sucesso via `buscarOrcamentoPorNumero`, o link não é gerado.

## Fix

Remover a guarda `isAssociatedCustomer` das duas ocorrências da condição em `enviarParaCliente`, mantendo apenas `clienteId` como condição:
- Linha ~1801: geração do approval token e link
- Linha ~1842: observação de nome associado

## Tasks

### Task 1 — Corrigir condição de geração do approval link

**File:** `apps/backend/src/modules/quotes/quotes.service.ts`

**Change 1 (linha ~1801):**
- Before: `if (clienteId && isAssociatedCustomer) {`

**Change 2 (linha ~1842):**
- Before: `if (clienteId && isAssociatedCustomer) {`
- After: `if (clienteId) {`

**Verify:** O approval link é gerado quando `buscarOrcamentoPorNumero` retorna `idcliente`, independente de `isAssociated`.

**Done:** As duas condições foram alteradas e commitadas.
