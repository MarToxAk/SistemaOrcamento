# Summary — 260514-001: Correção Link de Aprovação no Envio

## O que foi feito

Corrigido o bug onde o link de aprovação nunca era gerado/incluído na mensagem enviada ao cliente via Chatwoot.

## Root cause

Em `enviarParaCliente` (quotes.service.ts), duas condições bloqueavam a geração do approval link:
```
if (clienteId && isAssociatedCustomer)
```
`isAssociated` tem valor padrão `false` no modelo Customer e nunca era alterado para `true` automaticamente ao importar orçamentos do Athos. Mesmo que `clienteId` fosse resolvido com sucesso via `buscarOrcamentoPorNumero`, o link jamais era gerado.

## Fix aplicado

Removida a guarda `isAssociatedCustomer` nas duas ocorrências em `enviarParaCliente`:
- Linha 1801: geração do approval token e link → `if (clienteId)`
- Linha 1842: observação de nome associado → `if (clienteId)`

O approval link agora é gerado sempre que `idcliente` Athos é encontrado no lookup do orçamento.

## Commit

`fabc37e` — fix(quotes): gerar approval link sempre que idcliente Athos disponivel

## Impacto

- Orçamentos com idcliente Athos agora recebem o link de aprovação na mensagem
- Comportamento de aprovação do EM_PRODUCAO não é afetado (a checagem de `approvalRequestedAt` já estava presente como fallback)
- Nenhuma migração de banco necessária
