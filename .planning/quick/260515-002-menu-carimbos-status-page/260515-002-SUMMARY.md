# Summary - 260515-002: Menu de Carimbos na Página /status

## O que foi feito

Adicionado um novo seletor de filtro por carimbos (badges de pagamento) na página /status para acompanhar orçamentos por tipo de pagamento.

## Arquivo alterado

- `apps/frontend/src/app/status/page.tsx`

## Mudanças implementadas

1. Adicionadas constantes de tipos de carimbos:
   - BADGE_TYPES: array com tipos ["PAGO_CAIXA", "PIX_CONFIRMADO", "AGUARDANDO"]
   - BADGE_LABELS: mapeamento de rótulos em português

2. Criada função `getBadgeType(quote)`:
   - Determina qual tipo de carimbo exibir para cada orçamento
   - Lógica: Pago no Caixa > PIX Confirmado > Aguardando pagamento

3. Adicionado estado `selectedBadgeFilter` (padrão: "TODOS")

4. Modificada lógica de `visibleQuotes`:
   - Agora filtra por ambos Status E Carimbos
   - Usuário pode combinar filtros

5. Renderizado select na barra de cabeçalho:
   - Label: "Carimbos:"
   - Opções: Todos, Pago no Caixa, PIX Confirmado, Aguardando pagamento
   - Posicionado logo após filtro de Status existente

## Validação

- `npm --workspace @bomcusto/frontend run build` ✓ compilou com sucesso
- Sem erros de TypeScript
- Filters funcionam independentemente (Status vs Carimbos)

## Resultado

Usuário pode agora:
- Filtrar por um status específico
- Filtrar por um tipo de carimbo específico
- Combinar ambos os filtros para acompanhamento mais preciso
