# Summary — 260514-002: Notificação Caixa + PIX Chatwoot

## O que foi feito

Corrigidos dois bugs no fluxo de notificação de pagamento ao cliente.

## Bug 1: Pagamento no caixa não notificava o cliente

**Root cause:** O `AthosListenerService` recebia a notificação via PostgreSQL LISTEN/NOTIFY e persistia `paymentNote`, mas não chamava nenhuma lógica que enviasse mensagem ao cliente via Chatwoot. A notificação via Chatwoot só acontecia LAZILY quando alguém abria o orçamento no frontend (via `getById → conciliarViaCaixaAthos`).

**Fix:**
1. `QuotesService.confirmarPagamentoCaixa()` — novo método público que: persiste `saleExternalId`, `paymentNote`, `paymentConfirmedAt`; muda status para APROVADO via `changeStatus()` (com histórico); envia mensagem `✅ Pagamento confirmado no caixa! Pedido #X — Orçamento #Y` ao Chatwoot
2. `AthosListenerService` — injeta `QuotesService` via `forwardRef` e chama `confirmarPagamentoCaixa()` após emitir o SSE
3. `athos.module.ts` — importa `QuotesModule` via `forwardRef` (mesmo padrão de `EfiModule`)

## Bug 2: Mensagem PIX muito verbosa

**Root cause:** O EFI webhook enviava uma mensagem longa com lista de itens, PDF URL e pergunta de NF — informação desnecessária para uma confirmação de pagamento.

**Fix:** Mensagem simplificada:
- Pagamento completo: `✅ Pagamento PIX recebido! Orçamento #N — R$X. Comprovante gerado. Pedido confirmado!`
- Entrada 50/50: `✅ Entrada PIX recebida! Orçamento #N. Entrada: R$X — Restante: R$Y a pagar na loja.`
- Parcial outro: `⚠️ Pagamento parcial PIX recebido. Orçamento #N. Pago: R$X de R$Y — Restam: R$Z`
