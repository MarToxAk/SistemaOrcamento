## v1.2 Requirements

### Mensagens Automáticas ao Cliente (MSG)

- [x] **MSG-01**: Quando cliente aprova orçamento via token, Chatwoot envia mensagem com emoji confirmando aprovação e informando próximo passo do serviço
- [x] **MSG-02**: Quando pagamento PIX parcial é confirmado via webhook EFI, Chatwoot envia mensagem com valor recebido, saldo restante e status atual
- [x] **MSG-03**: Quando pagamento PIX total é confirmado via webhook EFI, Chatwoot envia mensagem de conclusão/celebração do pagamento completo
- [x] **MSG-04**: Quando pagamento parcelado é confirmado via webhook EFI, Chatwoot envia mensagem confirmando a parcela recebida e informando restante
- [x] **MSG-05**: Quando status do orçamento muda para EM_PRODUCAO, PRONTO, ENTREGUE ou CANCELADO, Chatwoot envia mensagem informando o cliente com texto adequado para cada status

### UX das Páginas Públicas do Cliente (UX)

- [x] **UX-01**: Página de aprovação (`/orcamento/:id/approve`) tem layout visual consistente com identidade da marca Bom Custo e demais páginas do sistema
- [x] **UX-02**: Página de status (`/orcamento/:id/status`) exibe status atual do serviço de forma clara com ícone/emoji por status e descrição amigável em português
- [x] **UX-03**: Ambas as páginas têm aparência acolhedora, responsiva e com identidade coesa (logo, gradiente, card branco centralizado)
