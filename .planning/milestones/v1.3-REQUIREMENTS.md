## v1.3 Requirements (Archived)

### Migrations (MIG)

- [x] MIG-01: Operador pode atualizar os containers com Docker Compose sem erro de migration já aplicada
  - **Outcome:** Implementado via `prisma migrate deploy` (idempotente por design) com bootstrap-runtime.sh garantindo sequência correta
- [x] MIG-02: Backend executa migrations apenas quando o PostgreSQL estiver pronto para conexões
  - **Outcome:** Implementado via wait-for-db.js com polling + timeout configurável antes de qualquer step de migration
- [x] MIG-03: Em falha de migration, logs exibem causa técnica e comando de ação recomendado
  - **Outcome:** bootstrap-runtime.sh exibe `MIGRATION_DEPLOY_FAILED` com exit code não-zero; runbook documenta ação de resposta
- [x] MIG-04: Fluxo de startup não entra em loop de restart por falha transitiva de banco não pronto
  - **Outcome:** wait-for-db.js aguarda banco e encerra com `process.exit(1)` após timeout — Docker não reinicia indefinidamente

### Operação de Deploy (OPS)

- [x] OPS-01: Existe runbook de update com passos determinísticos para VPS (pull, up, verify)
  - **Outcome:** `deploy/UPDATE_RUNBOOK.md` entregue com seções Pre-check, Update passo a passo, Validação, Rollback tático e Troubleshooting
- [x] OPS-02: Existe procedimento de verificação pós-deploy para confirmar schema e saúde da API
  - **Outcome:** `scripts/verify-deploy-health.ps1` verifica compose status, logs críticos e health endpoint HTTP 200
