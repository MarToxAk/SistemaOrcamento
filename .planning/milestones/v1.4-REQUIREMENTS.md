## Requirements Completed

### Webhook EFI
- [x] **EFIW-01**: Endpoint de webhook EFI aceita notificacao sem exigir assinatura HMAC obrigatoria
- [x] **EFIW-02**: Webhook continua com protecoes de resiliencia (throttle, idempotencia por eventId/txid)
- [x] **EFIW-03**: Eventos de pagamento recebidos sao persistidos com metadados suficientes para auditoria

### Conciliacao Athos
- [x] **ATHP-01**: Implementar `verificarPagamentoPorOrcamento` no AthosService com consulta real ao Athos
- [x] **ATHP-02**: Checagem de pagamento via Athos considera `orcamentoId` e `vendaId` quando disponivel
- [x] **ATHP-03**: Quando pagamento confirmado no Athos, status do orcamento e atualizado automaticamente no fluxo permitido

### Gatilhos de Verificacao
- [x] **PCHK-01**: Ao abrir detalhe de orcamento, sistema dispara checagem de pagamento no Athos
- [x] **PCHK-02**: Ao enviar orcamento para cliente, sistema dispara checagem de pagamento no Athos
- [x] **PCHK-03**: Endpoint de status de pagamento retorna dados de conciliacao e resultado de sincronizacao de status

### Operacao e Observabilidade
- [x] **OBSV-01**: Logs estruturados registram resultado de cada tentativa de conciliacao (sucesso, nao pago, erro)
- [x] **OBSV-02**: Falhas de leitura no Athos nao derrubam o fluxo principal de orcamentos (degradacao graciosa)

### NFS-e Desconto
- [x] **NFSD-01**: Emissao de NFS-e aceita campo booleano para ativar/desativar desconto
- [x] **NFSD-02**: Emissao de NFS-e aceita desconto por percentual e calcula valor com base no total pago informado
- [x] **NFSD-03**: Emissao de NFS-e aceita desconto por valor fixo em reais
- [x] **NFSD-04**: Regras impedem desconto invalido (negativo, maior que base, percentual acima de 100)
- [x] **NFSD-05**: XML da NFS-e preenche campo de desconto coerente quando a flag estiver ativa
