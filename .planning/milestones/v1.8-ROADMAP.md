# Milestone v1.8: Aprovacao Associada ao Pagamento + Conciliacao Caixa Athos

**Status:** SHIPPED 2026-05-05
**Phases:** 19
**Total Plans:** 2

## Overview

Garante que aprovacao/entrada em producao so ocorra para orcamentos associados ao cliente Athos
(idcliente valido) com pagamento confirmado. Implementa conciliacao via `relacao_orcamento_venda`
ao abrir o orcamento, exibe indicador de "Pago no Caixa - Venda #N" na pagina publica e notifica
o cliente via Chatwoot quando o pagamento for confirmado no caixa.

## Milestone Summary

**Key Decisions:**

- Guard `EM_PRODUCAO` exige `isAssociated=true` (cliente Athos valido) E (`approved=true` OU `saleExternalId` presente)
- Fallback Athos por `idcliente`: mesmo quando `isAssociated=false` no campo, se aprovado por link
  (`approvalRequestedAt/approvalToken/approvalExpiresAt` presentes), detecta associacao via `idcliente > 0`
- Mensagens de erro em ASCII puro (sem acentos) para evitar problemas de encoding
- `conciliarViaCaixaAthos` usa fire-and-forget via `void ... .catch()` para nao bloquear `getById`
- Persistencia de `saleExternalId` idempotente: so atualiza quando campo ainda null

**Issues Resolved:**

- Guard EM_PRODUCAO nao bloqueava clientes nao associados sem pagamento (bug pre-v1.8)
- Textos de erro com mojibake (encoding incorreto em mensagens de aprovacao)
- Falta de visibilidade do pagamento via caixa na pagina publica do cliente
- Ausencia de notificacao Chatwoot para pagamentos confirmados via Athos/caixa

**Issues Deferred:**

- Trigger via LISTEN/NOTIFY PostgreSQL para eventos Athos em tempo real (ficou no Backlog)
- Relatorios e exportacao CSV (movido para v1.9 / Phase 20)
