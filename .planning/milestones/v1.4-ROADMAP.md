## Milestone Summary

### Key Decisions
| Date | Decision | Reason |
|------|----------|--------|
| 2026-05-04 | Nao usar n8n para notificacao de pagamento | Fluxo deve ficar na aplicacao backend principal |
| 2026-05-04 | Checar pagamento no Athos ao abrir e ao enviar orcamento | Sincronizar status com estado real do caixa |
| 2026-05-04 | Incluir desconto opcional na emissao de NFS-e | Necessidade fiscal de deducao por percentual/valor sobre total pago |
| 2026-05-04 | Webhook EFI sem HMAC obrigatorio mas com throttle + idempotencia | Reducao de atrito operacional com hardening minimo mantido |

### Delivered Features
- Webhook EFI aceita payload sem `x-signature`/`x-gn-signature`; throttle + idempotencia mantidos
- `AthosService.verificarPagamentoPorOrcamento` consulta tabelas reais via dynamic column discovery
- Fire-and-forget de checagem de pagamento em `getById` e `enviarParaCliente`
- Log estruturado `conciliacao_athos quoteId=... paid=... statusUpdated=...`
- `EmitirNfseInput` com `descontoAtivo`, `descontoPorcentagem`, `descontoValor`, `totalPago`
- `DescontoIncondicionado` populado dinamicamente no XML RPS
