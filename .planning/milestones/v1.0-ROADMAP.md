# Milestone v1.0: MVP — Segurança, Confiabilidade e UX

**Status:** ✅ SHIPPED 2026-05-02
**Phases:** 1–5
**Total Plans:** 8 planos executados

---

## Overview

Primeira versão completa do Sistema de Orçamento BomCusto com autenticação, observabilidade de integrações, correções de fluxo de dados, cobertura de testes e UX do painel interno e área do cliente.

O sistema partiu de um estado brownfield (zero testes, zero autenticação, integrações silenciosamente falhando) e chegou a um estado com guards globais, webhooks validados, testes automatizados em CI, e páginas de aprovação/status para o cliente.

## Milestone Summary

**Stats:**
- Fases: 5 (1–5)
- Planos: 8 executados
- Testes: 32 automatizados
- CI: GitHub Actions configurado
- Timeline: 2026-04-07 → 2026-05-02 (25 dias)
- Commits: ~30 (git log conta ~30 do brownfield ao milestone)
- Arquivos modificados: 134 (+22.782 / -1.681 linhas)

**Key Decisions:**

| Decisão | Racional |
|---------|----------|
| Guard global com `x-internal-api-key` em vez de OAuth | Equipe usa Chatwoot; sem cadastro próprio |
| Fire-and-forget para `enviarParaCliente` | Evita timeout HTTP em operações de 5 etapas síncronas |
| `pg.Pool` para Athos (max 5) | Previne exhaustion de conexões em requests concorrentes |
| BullMQ não implementado | Substituído por fire-and-forget in-process (menor infra) |
| `isAssociated` como campo booleano real | Magic string `__associated__` em `notes` era frágil e invisível |

**Issues resolvidos durante o milestone:**

- NestJS v10/v11 mismatch causava 500 em vez de 401 em todos os guards
- `isAssociated` não retornado pelo `mapQuoteBody()` — invisível na API
- `pg.Client` por request causava connection leak no Athos
- `enviarParaCliente` bloqueava a thread HTTP por até 30s em casos lentos

**Tech debt conhecido:**

- `quotes.service.ts` ~1800+ linhas — god class, extração gradual planejada
- Nenhum `as any` novo introduzido mas os existentes permanecem
- Fase 6 (integração Athos para envio automático) ainda pendente
- UAT da Fase 2 (runtime) não completado (testes de código passam)
